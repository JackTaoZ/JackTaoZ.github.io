CMU-GO使用手册

发表于 2018-08-28 | 分类于 RF Test Tool |

简介

  CMU200是测试GSM,EDEG,WCAMA,HSPA的RF的专用仪器，CMU GO是其自动化测试的软件。该软件只支持2/3G耦合测试，支持全网通的需要CMW-RUN作为自动化测试软件，设备也需要使用CMW500。本章节主要讲解CMU GO的使用配置和操作方法。

软件安装

  1、该软件安装简单，直接从网下下载软件后安装即可,软件界面如下图，安装完成需要搭建CMU200和屏蔽箱环境即可开始测试,首次安装需要安装GPIB卡驱动。安装包下载[CMU-GO]
  

脚本配置

  2、进入软件界面需要配置相关的脚本参数，然后以金机为基准，调试最佳参数文件。以GSM900/1800为例讲解相关配置过程。脚本设置完成，选择一个路径进行保存，因为机型不一样，所以每次测试的时候需要选择相应的脚本。load-sequence为选择脚本，save-sequence为保存脚本，每一次修改脚本必须保存，否则下次打开还是原始数据。
  

  3、配置脚本解读

  Basic Initializing：初始化配置

  GSM Call Setup：呼叫配置

  Delay：延时函数

  GSM Call Testset：信道Power/RX level配置

  GSM BER Search：末信道BER配置

  GSM Call release：呼叫释放断开

  • Basic Initializing：选择测试频段，可以选择GSM900/GSM1800/CDMA/WCDMA/Bluetooth等，第一项RF必须选上。

  • GSM Call Setup：按照图示依次设置相应经验参数，RF Level的参数直接关系呼叫连接速度，Attenuations根据实际设置。

  • Delay：延时参数设置，一般默认设置10。

  • GSM Call Testset：按照图示依次设置相应的频段、信道、功率等级，RF-Level的参数直接关系呼叫连接速度，Attenuations根据金样一步步经验调试。

  • GSM BER Search：末信道测试误码率，界面设置相关参数如图。

  • GSM Call release：测试项目完成，释放断开测试，整个测试完毕。

设备搭建

  配套设备：CMU200、屏蔽箱或者三角锥、射频连接线、测试天线、测试夹具、电脑一套、GPIB卡。
  
  

软件测试操作

  4、确认测试脚本

  4.1、查看当前使用的测试脚本是否正确，按如下步骤操作，点击步骤1图标，在弹出窗口中查看下图2处测试脚本是否为当前测试电话手表机型一致，如不一致按下步骤B.添加测试脚本。
  

  4.2、添加测试脚本，按下图示操作，单击打开步骤1，在弹出对话框中单击打开步骤2，在下拉中找到D盘根目录下的“GSG_CMUGO”脚本文件夹,在打开的GSG_CMUGO文件中选中对应机型的测试脚本如下图步骤3所示（xx1：xx1-QA-GSM900 and 1800.SEQ; xx2：xx2-QA-GSM900 and 1800.SEQ; xx3：xx3-QA-GSM900 and 1800.SEQ)，选中对应脚本后点击步骤4打开，测试脚本添加进来后点击步骤5 OK完成。

  5、开始测试

  5.1、按下图步骤1点击运行图标开始测试，首次测试会弹出步骤2窗口，点击“continue”即可进入正常测试；

  5.2、手机关机状态下装好白卡后开机，迅速将手机放入屏蔽箱测试夹具上等待呼叫响铃，呼叫铃声响后按接听，迅速关上屏蔽箱等待测试完成,测试完成后结果如下图：

  

  6、检查测试结果

  6.1、每天测试前先用金机测试一次，并记录测试结果。每测一台手机都要记录测试结果，检查测试结果是否在标准范围内，测试标准如下附件表格所示，测试完成需按照表格记录相关的数据。

  7、异常问题处理方法

  • 长时间呼叫不上，手机不响铃：终止CMUgo测试，将小机拿出看手机屏幕左上角是否显示有信号格数，如显示有信号格数，点击运行CMUgo,重启手机再放回去测试；如显示无信号，关机再重新放白卡，点击运行CMUgo，将手机开机后放到屏蔽箱继续等待呼叫。
  • 如以上方法还是无法测试则确认是否已绑定，切记绑定后的将无法连接测试。
  • 测试结果超出标准范围，联系此相关工程师现场手动测试确认。
