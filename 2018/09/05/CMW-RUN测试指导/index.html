CMW-RUN测试指导

发表于 2018-09-05 | 分类于 RF Test Tool |

脚本添加

  添加设置好的脚本进行测试。将脚本保存在英文路径的文件夹里（文件夹名称均以英文命名），点击add进行添加，如图所示：

  

设置线损

  点击Resources->Test setup添加线损文件，如没有现成的线损文件，需创建一个新的线损文件，点击*.fda files，创建新的线损文件。根据实际情况补充线损，传导根据金板调试线损，耦合测试一般设置成20即可，频率范围需覆盖800MHz~2600MHz。设置完后保存在相应的文件夹以便调用。如下图：第3步处，左边输入测试的频段，右边输入相应的线损。第4步保存，然后第5步，激活fda，第6步点击加载刚才所保存的线损文件。

  • 注意：步骤T2.2要确定已勾选上，否则脚本不执行线损补偿。
      

设置测试结果存储路径

  按下图步骤设置存储路径，先点击“Resource”在下拉菜单中选择“Measurement Report”,在Measurement Report界面中设置存储路径和测试结果保存格式（第三步：设置存储路径；第四步：勾选格式类型，一般选PDF格式）。
  

测试模式选择

  比如三个测试脚本GSM/WCDMA/LTE，每个测试脚本可以单独测试，也可以复合使用，如果只需测试一个脚本，则在测试脚本最后一条命令Testplan_Start栏下取消勾选Start，如果想自动继续下一个脚本的测试则双击此条命令添加下一个脚本。

点击测试

  • 脚本在运行时，可以暂停（Pause)或者直接终止(Abort)。
  • 脚本执行到步骤T2.4时，会弹出一个提示框，如下图所示，意思是需要终端进行一次开关机，当然开关一下飞行模式也是可行的。其目的是使仪器能够读到终端注册时相关的信息。
  • 在连接成功后，仪器会对终端进行呼叫（即步骤T2.8），如果是语音业务，会弹出一个响铃的对话框，请及时接听，如果是数据业务则无对话框。
  • 脚本执行到步骤T2.7时，会弹出如左下图。提示TAC-unknown。但不影响测试，可点击关闭，继续进行测试。如果不需要其弹出，则在本文第4页，步骤T2.7处，将其勾选掉。同理，如果不想执行那一步骤，可将其勾选掉即可。
  • 测试模式设置，有的小机不支持自动测试，需要在小机内部进行勾选设置。

  

测试报告预览

  • 屏幕下方有几个选项卡，有实时的运行LOG供我们查看。也有每一语句对应的SCPI命令。
  • 右图中红框中的Filter下拉可选择所保存的内容类型，可保存只有Failed的测试指标，也可全部保存。
  • 测试报告有几种文本格式供选择，建议选择PDF格式，因其内容与报告预览相同，所以方便查看。
  • 在“Summary”下面可以显示有多少个测试项目pass和fail。
      

添加信道

  • 通过脚本或者测试报告，可以知道：只测试了1-124信道，没有975-1023这部分额外的信道。如果想测试这些信道，有两个办法：要么更改参数配置栏中步骤4的信道号，要么就直接优化脚本。
  • 虽然优化脚本相对比较麻烦，但却是个一劳永逸的方法。优化后的脚本如下图，2个红框中的结构相同，不同的是步骤16中Loop循环值不同。
  • 其具体操作是，点击步骤4进行复制，然后再点击粘贴，此时就能粘贴出整个IF语句以及其从属语句。需要注意的是，逻辑关系不能混淆，对着步骤4粘贴，粘贴的内容便成为其根目录。从右图中的垂直虚线可知，上下红框的等级相同（同属于步骤4 IF语句下的2级目录）
  • 骤14和步骤17需要进行配置，否则参数配置栏会有感叹号提示（如右二图），脚本无法正常运行。
  • 步骤16的循环值需要更改为975-1023。其配置流程详见GSM脚本配置。

  

添加BER

  • 从结构上来看，它与TX相似，因其只需测最大功率等级，故没有PCL循环。
  • 从功能来看，测试语句不同，TX的是GSM_Testset，RX的是GSM_BERSearch。
  • 双击步骤24或29。可得下图：1是所测试的误比特率，2是测试的初始电平，3是测试的标准，低于此值，将会判定Failed。
  • 对于初始电平，为节省时间，建议将其调低。可根据经验调为终端的平均水平。
  • 所参考的原始脚本目录为：Installed-GSM- 可对其复制，再粘贴到你个人的脚本，操作步骤参考上文，此处不进行累赘。
  • 特别需要注意的是，更改原始脚本的内容后，切勿保存修改，否则便失去了其参考价值。
