CMW500 WIFI信令测试

发表于 2018-09-06 | 分类于 CMW500 |

简介

   CMW500仿真WLAN AP（SSID:CMW-AP）功能，手机作为用户终端接入该 AP，由 CMW500 完成手机发射和接收性能的测试。这里以 802.11b/g 1/7/13 信道为例，讲解WLAN信令测试的仪表和手机端配置。

环境搭建

  本节讲解CMW500手动测试WIFI Power/PER的操作过程，需要CMW500，射频线,屏蔽箱即可。CMW500相当于开启一个热点与手机进行连接，根据金机的天线方向性和OTA实际测试TRP/TIS值选择一个最佳位置，拦截不良坏机。

CMW500 设置&测试

  1、添加WLAN信令任务

  按SIGNAL GEN 按钮，进入 Generator/Signaling Controller 界面，选择 WLAN 功能中的Signaling选项，然后屏幕下方的任务栏菜单中会出现 WLAN Signaling功能菜单。该功能用于启动 WLAN AP 信令。如下图示：
  

  2、进入WLAN信令界面

  按任务栏菜单上的 WLAN Signaling 项，进入 WLAN 信令界面：
  

  3、配置 WLAN AP 信令参数

  以支持802.11b/g手机的测试为例，在信令界面更改如下图黄色圈中参数：

  在WLAN Signaling OFF状态下，配置如下参数：

  • Frequency/Channel为相应的高中低（1/7/13）信道；
  • Tx Burst Power 为仪表下行信号电平，设为－40.0dbm；
  • Expected PEP 为设置仪表测量最大发射期望功率，设为 25dbm。该值应比手机实际发射功率RX Burst Power高10到15db，因手机接入时发射WLAN信号是以最大功率发射，如果该值设置小于手机实际信号，仪表无法正确测量手机功率，会导致手机接入失败；
  • Standard 设置当前仪表和手机通信的标准，设为IEEE 802.11b/g；
  • Payload Size 设置发送数据包的数据大小，一般设为 500/1000。
      

  点击信令界面右下角“Config”，在弹出的界面下配置如上图黄色圈中的RF端口设置及外部衰减补偿。在 Connection 项下，ISI filter有 ype1，Type2 两种，该参数用于改善不同WIFI芯片厂商在802.11b 模式下的接入兼容性。在 802.11b 接入时如果手机端获取 IP 异常，将 WLAN Signaling off,尝试改变该参数来重新做接入。
  

  4、启动WLAN信令

  按ON/OFF按键，启动WLAN信令，WLAN信令启动完成后，界面如下图所示：
  

  5、开启手机WLAN，与CMW500 建立连接

  将手机WIFI信号通过RF线或天线耦合与 CMW500 连接，启动手机WIFI，将手机放在耦合箱里面，搜索到CMW_AP 后进行连接，该连接没有密码设置，接入成功后仪表界面如下

  • Connection Status 显示为 Associated；
  • Mobile Capabilities 显示手机的信息：
  • MAC 地址手机接入手机的 MAC 地址；
  • RX Burst Power手机发射的接入功率。为了让手机能顺利接入仪表，Expected PEP 应比该功率大10到15db 比较好。
      

  6、配置WLAN信令测试Multi Evaluation和PER 快捷方式

  将Packet Generator Configuration State 设为ON，用于手机TX和RX的测试；将信令界面的右边栏两个 Go To…键分别配置为 WLAN Multi Evaluation 和 WLAN PER，便于发射和接收测试界面的切换，然后如下图：

  

  通过config…按键可以设定或取消该快捷方式的设置，如下图所示：

  

  7、启动WLAN信令 TX测试Multi Evaluation

  在WLAN Signaling 界面按 WLAN Multi Eval.键，进入 WLAN Multi Evaluation 界面测试，按右上角 Multi Evaluation 键，然后按 ON/OFF 按键或 RESTART/STOP 按键，启动 WLAN 信令测试， Multi Evaluation 处于 RUN 状态，底色为黄色，接收手机发射的 WLAN 信号。

  

  在该界面下按“Assign Views”键可以设置测试项目的开关。

  

  在Trigger 选项中，设置测试信号的触发方式，实际测试时可以尝试使用 Free Run，IF Power 和 WLAN Sig1：RXFrameTrigger 三种方式中的某一种，便于测试到稳定的信号：

  

  在Input-Signal选项中，可以根据当前手机的工作模式来设置信号测试类型，便于仪表正确分析。

  

  8、WLAN PER测试

  按TASKS键，显示出任务栏，如下图：

  

  通过任务栏的 WLAN Signaling 键返回到 WLAN Signaling 界面，通过前面设置的 WLAN PER 快捷键，进入到 PER 测试界面，如下图：

  

  设置 CMW500下行信号电平TX Burst Power，信号类型 Traffic Burst，统计数据量 Packets 等参数后，按ON/OFF按键或RESTART/STOP按键，完成 WLAN PER 测试。针对于WIFI一般误包率（PER）要求小于10%，通过不断设置TX Burst Power值以达到最大下限。

  

数据分析

  9、WLAN Message Analyzer

  WLAN Signaling 中集成了 Message Analyzer 工具，它可以抓取仪表和手机之间的信令交互信息，用于故障分析和定位。如果需要使用 Message Analyzer，在 WLAN Signaling OFF 状态下，通过 Config…配置 Message Analyzer 需要的信息项目，如下图：

  

  在WLAN Signaling ON 状态下，通过 Message Analyzer 键打开 WLAN Analyzer 工具，开始监控手机和仪表间的信令信息，最后可以将该信息保存成文件供分析，如下图：
