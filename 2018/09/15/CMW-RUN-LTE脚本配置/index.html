CMW RUN LTE脚本配置

发表于 2018-09-15 | 分类于 RF Test Tool |

LTE脚本解读

  • 步骤2进行呼叫设置。
  • 因为模板类的测试无法统一测试，所以，将这4项单独分出来进行测试。
  • 图中2处选择TDD或FDD；4处勾选上。
  • 选择测试的BW，Band，和CH，以及测试的项目，双击1处可对测试信道进行配置。若选择Clear Channel 则不进行此带宽的测试，若选择Use Default则是默认信道。
  • 图中3设置电源的类型
  • 在LTE的测试过程中，程序会经常弹出对话框，提示对终端进行开关机。若终端没有重启或者开关飞行模式，程序会对端口通断一次，使终端重新注册，但这种被动的方法成功率不高。所以必需寻找可控的方案解放人手，提高效率。

  

电源控制

  可以通过对外部电源控制实现重启，点击电源类型设置后，如下图。

  • 图中1选择外部电源
  • 图中2选择相应的程控电源
  • 图中3选择通断的间隔
  • 部分终端在掉电，又瞬间上电后会重启。利用这个特性，当终端需要开关机时，程序会控制电源通断一次，这时终端便会重启并注册。
  • 图中1可选择AT Modem ….，前提是终端和PC有相关的环境。
  • 图中1也选择Customized Test Plan.意思是用户自定义的方案。

  

  终端还可以通过ADB命令的办法进行重启。当然，需要PC有ADB的环境，以及终端打开USB调试。

  • 在上图1处选择Customized Test Plan后，如下图，在红框处加载ADB命令的脚本
  • ADB命令的语句可单独保存为脚本，如下图，双击打开，输入重启的命令。
  • 外部电源控制的方法适用于：终端掉电，瞬间加电后可以重启，并且通过假电池外部供电的传导测试。
  • 若终端无法通断电重启，且需要整机测试，那么可采用ADB命令的方法。
  • ADB命令的方法稳定性不如外部电源，因为测试过程中终端有可能断开ADB链接。而外部电源则是强制性的。

  

BLER脚本设置

  • 不同的Band可通过硬切换的办法，即断开链接后，重新呼叫。
  • 步骤5处设置测试的Band,以及将CH的值进行导入。
  • 不同的BW间可通过步骤9的语句,进行软切换，即不需要断开链接。但需要配置目标BW以及RB数量和位置。
  • 双击步骤7，可见配置界面，配置初始电平及通过率限值。
  • 红框中的值并不是其灵敏度，只是其RS EPRE值，可通过公式进行转换：Full Cell BW Pow=RS EPRE + 10 lg(BW）125)
  • 此脚本无参考原始脚本，自行编写。

  

同项目/频段 多脚本测试衔接

  • 可以通过TestPlan_Start语句进行衔接。
  • 图中红框处选择下一个脚本的路径。在完成当前脚本后，将会打开并且直接开始测试下一个脚本，图中1选择是否要混合测试报告。

  

不同项目/不同频段 多脚本测试衔接

  不同频段，不同项目测试衔接完成后，需要飞行下或重启才能重新注册到新的网络。

  • 可以把ADB命令放在每个脚本的前面，重启完成后再进行呼叫。
  • 控制外部电源的通断进行重启。由于程控电源是一个SCPI 的设备，所以可以利用SCPICommandList这个语句对其控制，红框处选择电源的地址，在输入框中输入通断的SCPI命令。

注意事项

  • 在工程模式中将应答方式设置为自动接听。
  • 板级测试的话，设置好程控电源。
  • 整机测试设置好ADB环境。
  • world mode 设置为对应的制式。
